# 数字人大脑后端 - 完整使用文档

## 一、项目概述

本项目是一个**完全基于本地大模型**的数字人"大脑"后端服务，无需任何云端 LLM API。

核心能力：
- **RAG 知识库问答**：基于本地 PDF/TXT/Markdown 文档进行检索增强生成
- **Agent 工具调用**：自动调用天气查询、通知发送、时间获取等工具
- **语音交互闭环**：STT（语音转文字）→ 本地 LLM → TTS（文字转语音），通过 WebSocket 实时传输

技术栈：Ollama + LangChain + Chroma + Whisper + gTTS + FastAPI

## 三、快速开始（5 步运行）

### 第 1 步：安装 Ollama

```bash
# 访问 https://ollama.ai 下载安装 Ollama
# 安装完成后，拉取模型（推荐 qwen2:7b，中文能力强）
ollama pull qwen2:7b

# 验证安装
ollama list
ollama run qwen2:7b  # 交互测试，Ctrl+D 退出
```

### 第 2 步：准备 Python 环境

```bash
cd digital_human_backend

# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
source venv/bin/activate        # Linux / Mac
zhishiku\Scripts\activate           # Windows

# 安装依赖
pip install -r requirements.txt
```

### 第 3 步：配置环境变量

```bash
# 复制模板
cp .env.example .env

# 编辑 .env 文件（按需修改）
```

`.env` 配置项说明：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `OLLAMA_BASE_URL` | `http://localhost:11434` | Ollama 服务地址 |
| `LOCAL_LLM_MODEL` | `qwen2:7b` | 使用的 LLM 模型名称 |
| `EMBEDDING_MODEL_NAME` | `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2` | Embedding 模型（首次自动下载） |
| `VECTOR_STORE_PATH` | `./vector_store/chroma_db` | 向量库持久化路径 |
| `WHISPER_MODEL_SIZE` | `base` | Whisper 模型大小（tiny/base/small/medium/large） |
| `TTS_LANG` | `zh` | TTS 语言（zh=中文, en=英文） |

### 第 4 步：构建知识库向量索引

```bash
# 1. 将知识库文件放入 documents/ 目录
#    支持格式：PDF、TXT、Markdown
cp /path/to/your/documents/*.pdf documents/

# 2. 运行构建脚本
python scripts/build_vector_store.py

# 输出示例：
# [信息] 找到 3 个文件待处理
#   [OK] report.pdf: 12 页
#   [OK] notes.md: 5 段
# [信息] 共加载 17 个文档片段，开始分割...
# [信息] 分割完成，共 85 个文本块
# [向量库] 构建完成，共 85 个文本块 -> ./vector_store/chroma_db
# [完成] 向量库构建成功！
```

### 第 5 步：启动服务

```bash
# 确保 Ollama 正在运行
ollama serve  # 如果没有自动启动的话

# 启动后端服务
uvicorn server:app --reload --host 0.0.0.0 --port 8000

# 输出：
# INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
# INFO:     Started reloader process
# [STT] 正在加载 Whisper 模型: base ...
# [STT] Whisper 模型加载完成。
```

### 4.4 WebSocket 语音交互

连接地址：`ws://localhost:8000/voice`

**协议流程：**

```
客户端                          服务端
  |--- 连接 ws://.../voice ----->|
  |                              |
  |--- 发送音频(二进制) -------->|
  |                              | STT 识别
  |<-- JSON: stt_result --------| ("你说: 你好")
  |                              | Agent 处理
  |<-- JSON: text_reply --------| ("回复文本...")
  |<-- 音频(二进制, MP3) -------| TTS 合成
  |                              |
  |--- 发送文本 "你好" --------->|
  |<-- JSON: text_reply --------| 
  |<-- 音频(二进制, MP3) -------|
  |                              |
  |--- 发送文本 "RESET" ------->| 重置会话
  |<-- JSON: info --------------|
  |                              |
  |--- 发送文本 "STOP" --------->| 断开连接
```

**消息格式：**

发送类型 | 格式 | 说明
---|---|---
文本消息 | 纯文本字符串 | 直接对话或控制指令
音频数据 | 二进制 (bytes) | WAV/MP3/PCM 音频

返回类型 | 格式 | 说明
---|---|---
STT 结果 | `{"type": "stt_result", "text": "..."}` | 语音识别结果
AI 回复 | `{"type": "text_reply", "text": "..."}` | AI 回复文本
音频数据 | 二进制 (bytes) | TTS 合成的 MP3 音频
信息通知 | `{"type": "info", "message": "..."}` | 系统信息

### 5.2 各文件详细说明

#### 配置层

| 文件 | 作用 | 关键函数/类 |
|------|------|-------------|
| [config/settings.py](config/settings.py) | 加载 .env 配置，提供全局配置常量和工厂函数 | `get_embedding_model()` 返回 Embedding 模型实例；`get_llm()` 返回 ChatOllama LLM 实例 |

#### 核心业务层 (core/)

| 文件 | 作用 | 关键函数/类 |
|------|------|-------------|
| [core/loader.py](core/loader.py) | 多格式文档加载器，支持 PDF/TXT/Markdown | `load_document(file_path)` → `List[Document]` |
| [core/splitter.py](core/splitter.py) | 文本分割器，将长文档切分为适合向量化的块（chunk_size=800, overlap=100） | `split_documents(docs)` → `List[Document]` |
| [core/vector_store.py](core/vector_store.py) | 向量数据库管理：构建、加载、检索 | `create_vector_store(docs)` 构建并持久化；`load_vector_store()` 加载已有库；`get_retriever(vs, k=3)` 返回检索器 |
| [core/rag_chain.py](core/rag_chain.py) | RAG 问答链，基于检索结果 + Prompt 生成回答，支持返回来源文档 | `create_rag_chain()` 创建 QA 链；`query_rag(question)` 直接问答 |
| [core/tools.py](core/tools.py) | Agent 可调用的工具定义（使用 `@tool` 装饰器） | `get_weather(city)` 查询天气；`send_notification(msg, to)` 发送通知；`get_current_time()` 获取时间 |
| [core/agent.py](core/agent.py) | Agent 核心逻辑：意图分类 → 路由到工具/RAG/直接对话 | `DigitalHumanAgent` 类；`chat(user_input)` 处理用户输入 |
| [core/memory.py](core/memory.py) | 会话记忆管理，支持多轮对话上下文 | `SessionMemory` 类：`add_user_message()`、`add_ai_message()`、`get_messages()`、`clear()` |

#### 语音层 (voice/)

| 文件 | 作用 | 关键函数/类 |
|------|------|-------------|
| [voice/stt.py](voice/stt.py) | 语音转文字，使用本地 Whisper 模型（支持 tiny/base/small/medium/large） | `transcribe(audio_bytes, model_size)` → `str` |
| [voice/tts.py](voice/tts.py) | 文字转语音，使用 gTTS（支持中文/英文） | `synthesize(text, lang)` → `bytes`（MP3 格式） |
| [voice/audio_utils.py](voice/audio_utils.py) | 音频格式转换工具（PCM→WAV、临时文件管理） | `pcm_to_wav()`、`audio_bytes_to_wav_file()`、`cleanup_temp_file()` |

#### 服务层

| 文件 | 作用 | 关键内容 |
|------|------|----------|
| [server.py](server.py) | FastAPI 主程序，提供 REST API + WebSocket 端点 | `GET /` 服务状态；`POST /chat` 文本对话；`POST /reset` 重置会话；`WS /voice` 语音交互 |

#### 脚本

| 文件 | 作用 | 使用方法 |
|------|------|----------|
| [scripts/build_vector_store.py](scripts/build_vector_store.py) | 扫描 documents/ 目录，构建/更新向量库 | `python scripts/build_vector_store.py` |

## 八、常见问题排查

### Q1: Ollama 连接失败
```
错误: Connection refused http://localhost:11434
```
**解决：**
```bash
# 检查 Ollama 是否运行
ollama list
# 如果未运行，启动 Ollama
ollama serve
```

### Q2: 模型下载慢
**解决：**
- 使用代理加速下载
- 选择较小的模型：`qwen2:3b` 或 `qwen2:1.5b`
- 模型文件默认存储在 `~/.ollama/models/`

### Q3: Whisper 首次加载慢
**解决：**
- 首次加载需要下载模型文件（约 150MB for base）
- 之后会缓存到本地
- 可在 `.env` 中将 `WHISPER_MODEL_SIZE` 改为 `tiny`（约 75MB）加速

### Q4: 内存不足 (OOM)
**解决：**
- 使用更小的 LLM 模型（`qwen2:3b`）
- Whisper 使用 `tiny` 版本
- 增加系统内存或使用 swap

### Q5: 向量库构建失败
**解决：**
- 检查 `documents/` 目录下是否有支持的文件（PDF/TXT/MD）
- 检查文件编码是否为 UTF-8
- 查看错误信息，确认文件未损坏

### Q6: 工具调用返回描述而非真实结果
**解决：**
- 确保 `core/agent.py` 中的 `_parse_and_execute_tool` 方法正确解析 JSON
- 检查 Ollama 模型是否能正确输出 JSON 格式
- 可尝试换用推理能力更强的模型（如 `qwen2:14b`）

## 九、配置进阶

### 切换 LLM 模型

```bash
# .env
LOCAL_LLM_MODEL=llama3:8b       # 切换到 Llama3
LOCAL_LLM_MODEL=qwen2:14b       # 切换到 Qwen2 14B
LOCAL_LLM_MODEL=qwen2:3b        # 轻量级模型
```

### 添加自定义工具

编辑 `core/tools.py`，添加新工具：

```python
@tool
def my_custom_tool(param: str) -> str:
    """工具描述（LLM 会根据此描述判断何时调用）。"""
    # 你的实现
    return f"结果: {param}"
```

然后在 `TOOLS` 列表中注册：
```python
TOOLS = [get_weather, send_notification, get_current_time, my_custom_tool]
```

### 知识库增量更新

```bash
# 1. 将新文件放入 documents/ 目录
# 2. 重新运行构建脚本（会覆盖旧向量库）
python scripts/build_vector_store.py
```

## 十、生产部署建议

1. **使用 Gunicorn + Uvicorn Worker** 替代 `--reload` 模式
2. **添加认证**：WebSocket 连接添加 token 验证
3. **监控**：添加 Prometheus metrics 或日志收集
4. **容器化**：编写 Dockerfile，将 Ollama 和后端服务分离部署
5. **负载均衡**：多个后端实例 + Redis 共享会话状态
